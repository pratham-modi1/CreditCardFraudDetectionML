Stage 1: Preprocessing and Feature Understanding

1. Column Understanding

- Time:
  Represents the number of seconds since the first transaction.
  Useful because fraudulent transactions often occur in bursts or unusual time patterns.

- Amount:
  Represents the transaction value.
  Fraudulent transactions may involve unusually high or low amounts, but amount alone is not a strong indicator.

- Class (Target Variable):
  0 → Legitimate transaction
  1 → Fraudulent transaction

- V1 to V28 (PCA Features):
  Original sensitive data (card details, location, merchant, etc.) is transformed using PCA.
  Each feature is a combination of multiple original attributes.
  Example (conceptual):
  V1 = 0.3×location + 0.9×time + merchant + behavior patterns

  These features are:
  - Uncorrelated
  - Ordered by importance (variance)
  - Not directly interpretable but useful for ML models


2. Dataset Quality

- No missing values
- No categorical variables (already encoded using PCA)
- Dataset is clean and ready for modeling


3. Feature Importance Insight

Fraud detection is mainly based on behavioral patterns, not just transaction amount.

A ₹10 fraud is still fraud, and a ₹10,000 legitimate transaction is still valid.

Therefore, the model relies more on V1–V28 features than Amount.


4. Handling Duplicates

- Found 1041 duplicate rows and removed them

Reason:
- More duplicate fraud → bias towards fraud
- More duplicate non-fraud → bias towards normal class

Removing duplicates ensures unbiased learning.


5. Handling Skewness in Amount

- Amount column is highly right-skewed (many small values, few large ones)

Problem:
- Can negatively affect model performance

Solution:
- Apply log transformation using log1p

Example:
Original → After log
10       → 2.3
100      → 4.6
1000     → 6.9

Result:
- More balanced distribution


6. Feature Scaling

- Applied StandardScaler to Amount after log transform

Purpose:
- Prevent features with large values from dominating
- Ensure all features contribute equally


7. Time Feature Engineering

- Raw Time represents order, not meaningful directly

Fraud depends more on time of day than sequence.

Converted:
Hour = (Time / 3600) % 24

- Captures hour of transaction (0–23)
- Scaled Hour for consistency
- Dropped original Time column


8. Summary

- Removed duplicates
- Log-transformed and scaled Amount
- Converted Time to Hour and scaled it
- Dropped Time column

Dataset is clean and ready for EDA and modeling.

# Complete EDA Thought Process and Methodology (Fraud Detection)

## 1. Initial Thinking: Where to Start?

EDA should not begin randomly. The first step is to identify **intuitive and interpretable features**.

In this dataset:

* Amount → transaction value
* Time → converted to Hour

These are directly understandable, so they are analyzed first.

---

## 2. Analysis of Interpretable Features

### Amount

* Heavy overlap between fraud and legit
* Fraud more spread out
* High-value outliers mostly legit
  👉 Conclusion: Weak feature

### Hour

* Slight median shift (fraud earlier)
* Large overlap
  👉 Conclusion: Weak but useful signal

---

## 3. Key Realization

At this stage:

* No clear separation found
* Fraud is not obvious from simple features

👉 Insight:

> Fraud detection depends on **subtle patterns**, not obvious rules

---

## 4. Problem: How to Choose Next Features?

There are 28 PCA features (V1–V28).

❌ Random exploration = inefficient
✅ Need a data-driven approach

---

## 5. Solution: Correlation-Based Feature Selection

### Steps:

1. Compute correlation with target (Class)
2. Sort values
3. Select:

   * Top positive correlations
   * Top negative correlations

---

## 6. Interpretation of Correlation

* Positive correlation → higher value → more fraud
* Negative correlation → higher value → more legit

### Strength:

* 0.3+ → strong
* 0.1–0.3 → moderate
* 0.01–0.1 → weak but useful

👉 Even small values matter due to class imbalance

---

## 7. Why Correlation First?

* Reduces search space
* Focuses only on important features
* Avoids unnecessary plotting

👉 Moves EDA from random → strategic

---

## 8. Next Step: Distribution Analysis

Once features are selected:

👉 Understand HOW they behave

---

## 9. Choosing the Right Visualization

### Based on Data Type:

* Feature → Numerical
* Target → Categorical (binary)

---

### Plot Selection Logic:

#### Box Plot (Primary)

Used when:

* Comparing distributions across classes

Advantages:

* Shows median
* Shows spread (IQR)
* Shows overlap
* Easy comparison

---

#### Histogram (Secondary)

Used when:

* Understanding shape of distribution

Limitations:

* Hard to compare two classes directly

---

#### Scatter Plot

Not used because:

* Target is categorical
* No meaningful pattern visible

---

## 10. What to Look For in Box Plots

For each feature:

* Median shift
* Spread difference
* Overlap
* Outliers

---

## 11. Important Mindset Shift

❌ Not looking for:

* Perfect separation

✅ Looking for:

* Small shifts
* Probability changes

---

## 12. Feature-Level Analysis (Strong Features)

### V11

* Median: Fraud >> Legit
* Strong upward shift
* Good separation
  👉 Strong positive indicator

---

### V17

* Median: Fraud << Legit
* Strong downward shift
  👉 Strong negative indicator

---

### V14

* Median: Fraud << Legit
* Values heavily negative for fraud
  👉 Strong negative indicator

---

### V4

* Median: Fraud >> Legit
* Values higher for fraud
  👉 Strong positive indicator

---

## 13. Pattern Recognition

After analyzing multiple features:

👉 Clear pattern emerges:

* Some features increase fraud probability (V11, V4)
* Some decrease fraud probability (V17, V14)

---

## 14. Why Not Analyze All Features?

Because:

* Model will use all features anyway
* EDA goal = understanding, not exhaustion

👉 Stop when:

* Patterns become clear
* Insight saturates

---

## 15. Interaction Analysis

Skipped because:

* Features are PCA-transformed
* Already uncorrelated
* Interactions are not interpretable

---

## 16. Final Understanding from EDA

* Amount, Hour → weak signals
* PCA features → strong signals
* No single feature is perfect
* Fraud detection = combination of multiple signals

---

## 17. Key Learning

EDA is not about:

* Finding rules

EDA is about:

* Understanding relationships
* Identifying patterns
* Estimating feature importance

---

## 18. Transition to Modeling

After EDA:

👉 Do NOT create manual rules

Instead:

* Train model (Logistic Regression)
* Let model learn:

  * Feature weights
  * Combined effects

---

## 19. Final Insight

EDA builds intuition like:

* “V11 ↑ → fraud”
* “V17 ↓ → fraud”

Model converts it into:

👉 Mathematical probability function

---

## 20. One Line Summary

EDA identifies **what matters**,
Model learns **how it matters**.
